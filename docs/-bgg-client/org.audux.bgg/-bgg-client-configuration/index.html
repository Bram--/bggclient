BggClientConfiguration

data class BggClientConfiguration(var maxConcurrentRequests: Int = 10, var maxRetries: Int = 5, var retryBase: Double = 2.0, var retryMaxDelayMs: Long, var retryRandomizationMs: Long, var requestTimeoutMillis: Long, var failOnUnknownProperties: Boolean = true, var requestsPerWindowLimit: Int = 60, var requestWindowSize: Duration = 60.seconds)(source)

Configure the BGGClient request strategies i.e. how many concurrent requests are allowed and how should retying be done.

Specifies an exponential delay between retries, is done using retryBase, retryMaxDelayMs and retryRandomizationMs This is then calculated using the Exponential backoff algorithm: delay equals to retryBase ^ retryAttempt * 1000 + [0..randomizationMs]
